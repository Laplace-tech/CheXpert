# Baseline 01 — DenseNet121 / U-Ignore / Frontal-Only / Official 5 Tasks

---

## 1. 실험 목적

CheXpert-small 기반 흉부 X-ray 멀티라벨 분류에서, 공식 5개 병변을 대상으로 사용하는 **기준 baseline** 을 재확립한다.  
이번 실험의 목적은 최고 성능 경쟁이 아니라, 이후 비교 실험의 기준이 될 **재현 가능한 baseline** 을 확보하는 것이다.

---

## 2. 실험 설정

### 데이터셋
- Dataset: `CheXpert-small`
- Data root: `data/chexpert_small/raw`
- Labels:
  - Atelectasis
  - Cardiomegaly
  - Consolidation
  - Edema
  - Pleural Effusion
- View mode: `frontal_only`
- Uncertainty policy: `U-Ignore`

### 모델 / 학습
- Backbone: `DenseNet121`
- Pretrained: `True`
- Input size: `320`
- Batch size: `32`
- Optimizer: `Adam`
- Learning rate: `1e-4`
- Weight decay: `0.0`
- Pos weight: `True`
- `pos_weight_clip_max`: `8`
- AMP: `True`
- Channels-last: `True`
- Epoch budget: `10`
- Seed: `42`

### 모델 선택 기준
- 마지막 epoch 모델이 아니라 **validation loss가 가장 낮은 checkpoint (`best.pt`)** 를 baseline 모델로 사용한다.

## 5. Eval 결과

### 평가 설정
- Checkpoint: `best.pt`
- Aggregation: `study-level max`
- Valid studies: `200`

### 전체 성능
- Mean AUROC: `0.8811`
- Mean AUPRC: `0.7387`

### 클래스별 성능
| Label | AUROC | AUPRC |
|---|---:|---:|
| Atelectasis | 0.8028 | 0.6855 |
| Cardiomegaly | 0.8484 | 0.7476 |
| Consolidation | 0.9077 | 0.6449 |
| Edema | 0.9100 | 0.7328 |
| Pleural Effusion | 0.9365 | 0.8829 |

### 해석
- 전체적으로 baseline 성능은 충분히 의미 있는 수준이다.
- strongest class: **Pleural Effusion**
- weakest class: **Atelectasis**
- `Consolidation`은 AUROC는 높은데 AUPRC는 상대적으로 낮아 threshold 민감도가 큰 클래스일 가능성이 있다.

## 6. Threshold Tuning 결과

### 설정
- Input: `eval/study_predictions.csv`
- Search range: `0.05 ~ 0.95`
- Step: `0.01`
- Criterion: `F1`
- 방식: label-wise threshold selection

### Best thresholds
| Label | Threshold | F1 | Precision | Recall |
|---|---:|---:|---:|---:|
| Atelectasis | 0.46 | 0.7059 | 0.5893 | 0.8800 |
| Cardiomegaly | 0.11 | 0.7044 | 0.6022 | 0.8485 |
| Consolidation | 0.47 | 0.6316 | 0.5455 | 0.7500 |
| Edema | 0.34 | 0.7129 | 0.6102 | 0.8571 |
| Pleural Effusion | 0.37 | 0.7867 | 0.6860 | 0.9219 |

### Recommended infer thresholds
```text
0.46,0.11,0.47,0.34,0.37
```

### 해석
- 대부분 클래스에서 `0.5`보다 낮은 threshold가 선택되었다.
- 고정 threshold `0.5`는 현재 모델에서 다소 보수적인 operating point로 보인다.
- 특히 `Cardiomegaly = 0.11`은 매우 낮은 값으로, calibration mismatch 또는 낮은 score distribution 가능성을 시사한다.

## 7. Error Analysis 요약

| Label | Threshold | FP | FN | Precision | Recall | F1 |
|---|---:|---:|---:|---:|---:|---:|
| Atelectasis | 0.46 | 46 | 9 | 0.5893 | 0.8800 | 0.7059 |
| Cardiomegaly | 0.11 | 37 | 10 | 0.6022 | 0.8485 | 0.7044 |
| Consolidation | 0.47 | 20 | 8 | 0.5455 | 0.7500 | 0.6316 |
| Edema | 0.34 | 23 | 6 | 0.6102 | 0.8571 | 0.7129 |
| Pleural Effusion | 0.37 | 27 | 5 | 0.6860 | 0.9219 | 0.7867 |

### 해석
- 이번 baseline의 threshold는 전반적으로 **recall 지향적**이다.
- FN을 줄이는 대신 FP를 꽤 허용하는 operating point가 형성되었다.
- 특히 `Atelectasis`, `Cardiomegaly`는 FP가 상대적으로 많다.
- `Pleural Effusion`은 recall과 precision이 비교적 균형적이라 현재 baseline에서 가장 안정적이다.

## 8. 최종 판정

### 결론
이번 실험은 **baseline 1차 확립 완료**로 본다.

그 이유는 다음과 같다.
- 데이터 점검 완료
- 로더 점검 완료
- 학습 완료
- best checkpoint 선택 완료
- eval 완료
- threshold tuning 완료
- error analysis 완료

즉 이번 run은 단순히 학습을 한 번 돌린 것이 아니라, **재현 가능한 기준선 1세트**를 완성한 것이다.

### 단, 주의점
- 이 baseline은 성능은 준수하지만, threshold tuning 결과는 **recall 쪽으로 약간 기운 운영점**이다.
- validation set에서 threshold를 최적화했기 때문에 같은 validation에서의 binary metric은 다소 낙관적일 수 있다.
- calibration 평가는 아직 하지 않았다.
- 별도 test set 검증도 아직 하지 않았다.
